Full diagnosis: findings page

Why the left looks broken

The review HTML uses grid-template-columns: 1fr 80px on each card, which paints an 80-pixel reference column of the swipe image directly against the right edge of every generated ad.

Because there's no gap, the swipe reference reads like a foreign sliver baked into the ad. It isn't — it's the layout. The ad PNG itself is a clean single image (the right column).

The fix — paste this into Claude Code

One change to ad-review.html: drop the swipe-reference column from each card. Open Claude Code in your funnel project and paste:

In funnel-pages/ad-review.html, remove the swipe-reference column from each card so the review only shows the generated ad, not the swipe image next to it. Specifically: 1. Change the CSS rule .card-images from `grid-template-columns: 1fr 80px;` to just `display: block;` (or `grid-template-columns: 1fr;`) so the generated image takes the full card width. 2. In the JS that builds each card (CONCEPTS.forEach loop), remove the <div class="swipe-thumb"> element entirely. Keep the <img class="generated"> tag. 3. Remove the now-unused .swipe-thumb and .swipe-thumb-missing CSS blocks. 4. Leave everything else (Keep/Regenerate buttons, badges, counts, export JSON) exactly as it is. Save the file and confirm the change by opening it in a browser. Each card should show the generated ad full-width with no narrow strip on the right.
